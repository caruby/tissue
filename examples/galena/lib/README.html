Galena Cancer Center Tissue Bank Migrator example

This migration example demonstrates how to load the content of a hypothetical custom tissue bank into caTissue. The use case illustrates several common migration impediments:

  • Different terminology than caTissue
  • Different associations than caTissue
  • Incomplete input for caTissue
  • Denormalized input
  • Inconsistent input
  • Input data scrubbing
  • Aliquot inference
  • Pre-defined caTissue protocol
These migration features are described in the migration process below.

Input data

The hypothetical Galena Tissue Bank data resides in the data/galena.csv CSV file included in the example. The CSV file holds one row for each specimen. The fields are as follows:

MRN
Patient Medical Record Number
Initials
Patient name initials
Frozen?
Flag indicating whether the specimen is frozen
SPN
Surgical Pathology Number
Collection Date
Specimen date of collection
Quantity
Amount collected
